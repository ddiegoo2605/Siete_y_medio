Nuestro programa

¿Como lo hemos hecho?

Logica banca

La banca sigue una estrategia específica para decidir cuándo pedir cartas y cuándo plantarse. Esta lógica se implementa en la función. La banca pedirá cartas hasta que su puntuación alcance o supere 7.5 puntos.

Logica bot

El bot sigue una estrategia predeterminada basada en la probabilidad. La función decide las acciones del boot basándose en su puntuación actual y las cartas visibles de la banca.

Prioridad

La prioridad inicial de los jugadores se determina al azar al comienzo de cada partida. La función asigna una carta aleatoria a cada jugador, y luego los ordena en función de estas cartas (numero/tipo) para establecer el orden de juego. Esta prioridad se mantiene durante toda la partida.

Insercion BBDD

Para registrar los datos del juego en la base de datos (BBDD), se utiliza la función. Esta función se conecta a la base de datos y ejecuta una consulta SQL que inserta los datos relevantes, como la identificación del jugador, la puntuación final y el resultado del juego. La conexión a la base de datos se maneja utilizando una biblioteca específica para la base de datos utilizada (por ejemplo, MySQL, PostgreSQL).
